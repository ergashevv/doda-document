1-bosqich onboarding — reja bo‘yicha xarita

Manba: onboarding_flow_tree rejasi. Asosiy g‘oya: 1 va 2.1 qadam hamma uchun bir xil, keyin uchta alohida yo‘l (Doda kuryer · Lavka · Eats). Lavka bilan Eats bir xil zanjir.

Sahifada nima bor
  1. 1-qadam va 2.1-qadam (hamma uchun)
  2. 2.1 dan keyin uchta parallel yo‘l
  3. Rejadagi umumiy daraxt (Mermaid)
  4. Yo‘l 1: Ru Park Doda kuryer (kod oqimi)
  5. Yo‘l 2 va 3: Lavka va Eats (bir xil TZ)

Boshida — faqat ikkita qadam (hamma uchun)

1-qadam. Ekranda: «Men parkda emasman (Doda taxi), ulanish kerak» ga o‘xshash matn. Foydalanuvchi Keyingi bosadi.

2.1-qadam (alohida ekran). Savol: «Qaysi servisda ishlashni xohlaysiz?» — uchta tugma:

  1. Ru Park Doda kuryer
  2. Yandex Lavka
  3. Yandex Eats

Shu joydan keyin oqimlar ajraladi — pastdagi Doda bo‘limi va Lavka bo‘limi ketma-ket emas, balki tanlangan tugmaga qarab parallel yo‘llar.

2.1 dan keyin — uchta parallel yo‘l

Bosqich 2.1 dagi tugmani tanlasangiz

1) Ru Park Doda kuryer

Transport → (faqat legkovoy: STS yoki 2× texnik pasport) → shahar → «RF fuqarosimisiz?» → dodaDocSequence → review.

2) Yandex Lavka

Shahar (Moskva / SPb) → fuqarolik (UZ/TJ, KZ/KG, RF, TM) → hujjatlar daraxti. Kod oqimi Doda kuryer kabi emas.

3) Yandex Eats

Lavka bilan to‘liq bir xil ketma-ketlik; farq faqat saqlanadigan servis nomi.

Lavka va Eats: shahar + fuqarolik + hujjatlar — mahsulot TZ. Doda kuryer: alohida modul (RF faqat boolean + transport/hujjat zanjiri).

Yo‘l 1: Ru Park Doda kuryer (ketma-ketlik + jadval)

Quyidagi tartib rejada yozilganidek: transport → (faqat car: mashina hujjati) → shahar → RF fuqarosi → dodaDocSequence.

1. Transport turi: car · truck · bike · foot · moto.

2. Faqat legkovoy (car): RF avto — STS; chet el — texnik pasport oldi + orqa. Boshqa transport turlarida bu blok yo‘q, to‘g‘ri shaharga o‘tadi.

3. Shahar (bk_G).

4. «Rossiya fuqarosimisiz?» (bk_Z, rfCitizen).

5. Hujjatlar — dodaDocSequence (quyidagi jadval). Pasport tugagach → bk_review.

categoryKey Shart Ketma-ketlik (kalitlar)
car vehicleRf === true license → sts → passport
car vehicleRf === false license → tech_passport_front → tech_passport_back → passport
truck — license → sts → gabaritlar → yuk kg → yukchilar → okleyka → passport
bike samozanyat yo‘q self_employed → thermal → passport
bike samozanyat ha self_employed → inn → thermal → passport
foot, moto — license → sts → passport

Qisqa ma’nolar: license → VU; sts → transport guvohnomasi oldi; yuk mashinasi qatorida — gabaritlar, yuk ko‘taruvchanlik, yukchilar, okleyka.

Yo‘l 2 va 3: Yandex Lavka va Yandex Eats (bir xil)

Avval: shahar (Moskva yoki SPb) → fuqarolik guruhi. Keyin tanlangan guruhdan kelib chiqib quyidagi tarmoqlar ochiladi.

Fuqarolik: O‘zbekiston yoki Tojikiston — hujjat turi

Patent:

  1. Pasport (foto sahifa)
  2. Patent oldi va orqa
  3. Registratsiya yoki Amina (reg: 2 rasm; Amina: 1 skrinshot)
  4. Migratsion karta
  5. Patent: rasmiy sahifa va oxirgi to‘lov (rasm + fayl)
  6. Rekvizit (video + rasm + matn) → 16 raqam → kartaga ulangan telefon → tugadi

VnZh: VNЖ oldi/orqa → migratsion karta → keyin RF bilan bir xil oxir: aloqa telefoni → rekvizit → 16 raqam → kartaga ulangan telefon.

Student: Student bileti → spravka → pasport → registratsiya yoki Amina → migratsion karta → rekvizit → 16 raqam → kartaga ulangan telefon.

Fuqarolik: Qozog‘iston yoki Qirg‘iziston

Pasport yoki ID tanlovi → pasport: faqat foto sahifa; ID: oldi va orqa → migratsion karta → propiska/registratsiya yoki Amina → rekvizit → 16 raqam → kartaga ulangan telefon.

Fuqarolik: RF

Pasport yuz/foto sahifasi → pasportdagi propiska → aloqa telefoni → rekvizit (video + matn + rasm) → 16 raqam → kartaga ulangan telefon.

Fuqarolik: Turkmaniston

Pasport foto sahifasi → ish yoki student vizasi tanlovi (ikkala turda ham keyin viza rasmi) → registratsiya yoki Amina → migratsion karta → aloqa telefoni → rekvizit → 16 raqam → kartaga ulangan telefon.
