Agar «Ru Park Doda kuryer» tanlansa

Doda · 1
Transport turi

Ekranda kategoriya tanlovi: legkovoy (car), yuk (truck), moto, velosiped (bike), piyoda (foot).

↓
Doda · 2 (faqat legkovoy)
Mashina hujjati — shartli
Agar legkovoy (car) tanlangan bo‘lsa
Keyingi ekran: Rossiyada ro‘yxatdan o‘tgan bo‘lsa — STS; chet el avto bo‘lsa — texnik pasport oldi + orqa (2 surat).
Agar truck / moto / bike / foot tanlansa
Ushbu qadam o‘tkazib yuboriladi → to‘g‘ridan-to‘g‘ri shahar tanloviga o‘tadi.
↓
Doda · 3
Shahar

Shahar tanlovi (kodda bk_G). Keyingi ekran: fuqarolik savoli.

↓
Doda · 4
«Rossiya fuqarosimisiz?»

Ha / yo‘q (bk_Z, rfCitizen). Keyingi: hujjatlar zanjiri dodaDocSequence (flow.js).

↓
Doda · 5 — hujjatlar (kategoriyaga qarab)
car + Rossiya mashinasi (vehicleRf = true)
Haydovchilik guvohnomasi (VU) → STS → pasport (zagran) → …
car + chet el mashina
VU → texnik pasport oldi → texnik pasport orqa → pasport.
truck
VU → STS → yuk mashinasi: gabaritlar → yuk ko‘taruvchanlik → yukchilar soni → okleyka → pasport.
bike
Samozanyatost’ savoli → agar ha: INN → termokorob → pasport; agar yo‘q: termokorob → pasport.
foot yoki moto
VU → STS → pasport.
↓
Doda · oxirgi
Review va yuborish

Pasport yuklangach — tekshiruv ekrani (bk_review), so‘ng arizani yuborish.

Agar «Yandex Lavka» yoki «Yandex Eats» tanlansa (ikkala servis — bir xil ketma-ketlik)

Lavka/Eats · 1
Shahar

Moskva yoki Sankt-Peterburg. Keyingi ekran: fuqarolik guruhi.

↓
Lavka/Eats · 2
Fuqarolik

Guruh tanlanadi: O‘zbekiston / Tojikiston, Qozog‘iston / Qirg‘iziston, RF, Turkmaniston. Keyingi zanjir tanlangan guruhga bog‘liq (pastda).

↓

O‘zbekiston yoki Tojikiston — avval «hujjat turi»

UZ/TJ
Hujjat turi: Patent / VnZh / Student
Patent tanlansa
Pasport (foto sahifa) → patent oldi/orqa → registratsiya yoki Amina (reg: 2 rasm; Amina: 1 skrinshot) → migratsion karta → patentning rasmiy sahifasi va oxirgi to‘lov (rasm + fayl) → rekvizit (video + rasm + matn) → 16 raqam → kartaga ulangan telefon → tugadi.
VnZh tanlansa
VNЖ oldi va orqa → migratsion karta → keyingi qadamlar RF bilan bir xil oxir: aloqa telefoni → rekvizit (video + matn + rasm) → 16 raqam → kartaga ulangan telefon → tugadi.
Student tanlansa
Student bileti → spravka (o‘qish) → pasport foto sahifasi → registratsiya yoki Amina → migratsion karta → rekvizit → 16 raqam → kartaga ulangan telefon → tugadi.
↓

Qozog‘iston yoki Qirg‘iziston

KZ / KG
Pasport yoki ID
Pasport
Faqat foto ko‘rinadigan sahifa.
ID karta
Oldi va orqa suratlari.

Keyin (ikkala yo‘lda ham): migratsion karta → propiska/registratsiya yoki Amina (reg: oldi/orqa; Amina: 1 skrinshot) → rekvizit (video + rasm + matn) → 16 raqam → kartaga ulangan telefon → tugadi.

↓

Rossiya fuqaroligi (RF)

RF
Ketma-ketlik

Pasport (yuz/foto sahifasi) → pasportdagi propiska sahifasi → aloqa uchun telefon → rekvizit (video + matn + rasm) → karta 16 raqami → kartaga ulangan telefon → tugadi.

↓

Turkmaniston

TM
Pasport va viza

Pasport foto sahifasi → ish vizasi yoki student vizasi tanlovi (ikkala turda ham keyin viza rasmi) → registratsiya yoki Amina → migratsion karta → aloqa telefoni → rekvizit karta → 16 raqam → kartaga ulangan telefon → tugadi.

Eslatma: Lavka va Eats uchun barcha qadamlar bir xil; farq faqat saqlanadigan service_type (lavka / eats) va hisobotda qaysi servis tanlanganligi. Doda kuryer esa alohida modul: transport + (faqat car uchun STS/texnik pasport) + shahar + RF fuqarosi savoli + dodaDocSequence.
