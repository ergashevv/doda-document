4. dodaDocSequence — kodda ketma-ketlik (src/flow.js)

Avvalo (agar fuqarolik rf, kz yoki kg): self_employed → agar «ha» bo‘lsa reg_amina → moy_nalog_phone. Keyingi qadamlar transportga qarab (pastdagi 4 ta blok).

Muhim: handlers.js dagi bk_SE:* (samozanyat javobi) faqat categoryKey ∈ { bike, car, truck } uchun ishlaydi — moto va foot uchun ketma-ketlikda self_employed qolsa, callback mos kelmasligi mumkin (ketma-ketlik va handlerlar nomuvofiqligi).

flowchart TB
  subgraph ct [Car yoki Truck]
    A1[SMZ + ixtiyoriy reg/moy_nalog] --> A2[vehicle_rf_pick]
    A2 -->|RF avto| A3[license → sts]
    A2 -->|chet el| A4[license → texnik old/orqa]
    A3 --> T0{yuk mashina?}

  end

  end

Izoh: dodaDocSequence bo‘yicha velo/piyoda uchun VU va STS yo‘q; moto uchun vehicle_rf_pick yo‘q, faqat VU + pasport.
